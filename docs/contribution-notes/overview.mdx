# Contribution Notes

This section is the knowledge base for contributors to the CloudFoundry Crossplane provider. It collects architectural decisions, implementation patterns, and essential background knowledge that goes beyond what the code itself communicates.

Whether you are onboarding as a new contributor or revisiting a non-obvious design choice — this is the place to look first.

## What you'll find here

- **Architecture Decision Records (ADRs):** Documented reasoning behind significant design and implementation decisions, so the "why" is never lost.
- **Best practices:** Patterns and conventions established by the team for consistency and maintainability across the provider.
- **Developer guides:** Focused how-tos for specific technical areas — covering setup, testing, and provider-internal mechanisms.

## Guides

- [Provider Overview](/docs/crossplane-provider-cf/docs/contribution-notes/provider-overview) — An overview of the Cloud Foundry provider.
- [Adding New Resources](/docs/crossplane-provider-cf/docs/contribution-notes/adding-new-resources) — How to add new resources to the provider.
- [Resource Names](/docs/crossplane-provider-cf/docs/contribution-notes/resource-names) — An introduction on how resources names are handled by the provider.
- [Upgrade Tests](/docs/crossplane-provider-cf/docs/contribution-notes/upgrade-testing) — How to run and interpret the upgrade test suite.
